Why LiteView2 Exists

The old WebBrowser control was built for a web that no longer exists.

❌ The Problem

VBA's WebBrowser Control Is Obsolete

  • 💀 Uses Internet Explorer — officially retired by Microsoft
  • 🚫 Cannot render modern HTML5, CSS3, or Flexbox layouts
  • ⚠️ Broken JavaScript — no ES6, no Promises, no Fetch API
  • 🔒 Many modern websites refuse to load entirely
  • 📉 No security updates — known vulnerabilities remain unpatched
  • 🤷 No two-way communication between VBA and JavaScript
✅ The Solution

LiteView2 — The Modern Replacement

LiteView2 embeds Microsoft's official WebView2 engine (the same Chromium engine that powers Microsoft Edge) directly inside your VBA forms. Every modern website renders perfectly.

  • ⚡ Full ES6+ JavaScript, CSS Grid, Flexbox, Web Animations
  • 🔗 Bidirectional VBA ↔ JavaScript messaging (3 channels)
  • 📊 Push DAO/ADO recordsets directly to web pages
  • 📦 Zero-install deployment — no admin rights needed
  • 🔄 Automatic security updates via Edge Evergreen runtime
  • 🛠️ 185+ APIs designed specifically for VBA developers

Two Ways to Deploy. Zero Headaches.

Up and Running in 3 Lines

No configuration files. No registry entries. No dependencies to manage.

VBA — Reg-Free Quick Start ' Create a browser pool — no regsvr32, no admin rights Set lv = LiteView2_CreateBrowserPool() ' Embed a Chromium browser inside your VBA form idx = lv.CreateInControl("https://example.com", Me.Frame1) ' Execute JavaScript and get the result back in VBA Dim title As String title = lv.ExecuteScript(idx, "document.title") MsgBox "Page title: " & title
VBA — Push Database Records to a Chart.js Dashboard ' Open a recordset from your Access database Dim rs As DAO.Recordset Set rs = CurrentDb.OpenRecordset("SELECT Period, Revenue FROM Sales") ' Send it to the browser — serialized as JSON, received by JavaScript lv.PushRecordset idx, rs, "renderChart" rs.Close
